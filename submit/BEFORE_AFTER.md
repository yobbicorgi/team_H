# Before / After — 수작업 → 에이전트화

## 우리가 자동화한 반복 수작업
지진해일 수치모델(ADCIRC 계열) 수행을 위한 10단계 전처리·실행 준비 과정. 작업자가 각 단계 폴더(`01_Deform_Plane` ~ `10_SLR_RUN`)에 직접 들어가 파일·코드·파라미터를 수동 수정하고 순차 실행하던 반복 작업을, 웹에서 파라미터만 설정하면 자동으로 전 단계를 수행/산출하도록 자동화.

## Before (기존 수작업 절차)
1. 시나리오(지진원/단층 파라미터, 해수면 상승 SSP 시나리오 등)마다 10개 단계 폴더를 순서대로 연다.
2. 각 단계 Python 스크립트(`Deform_plane.py`, `Tsu_xyz.py`, `01_Fort14.py`, `MSL_to_AHHW_*.py`, `Fort13_manning.py`, `Transport.py`, `SLR_Scenario.py`, `fort14_Create.py`, `fort14_Change_Elevation.py`, `Tsunami_SLR_RLT_CASE.py` 등)를 열어 입력 경로·케이스 번호·파라미터를 손으로 수정한다(스크립트마다 `_2/_3/_4` 케이스별 사본을 따로 둘 정도로 변형이 잦음).
3. 단계마다 수동 실행 → 산출물 확인 → 다음 단계 입력으로 수동 연결. 여러 시나리오를 돌리려면 이 과정을 시나리오 수만큼 전부 반복.
4. 결과는 단계 폴더 곳곳에 흩어져 있어 시나리오 간 비교가 번거로움.

## After (에이전트화된 절차)
1. 웹 플랫폼 좌측에서 시나리오 파라미터를 설정(또는 에이전트 채팅으로 셋팅값 지정).
2. "실행"을 누르면 파라미터가 단계별 설정값으로 변환되어 10단계가 자동 순차 수행(데모는 기존 결과/Mock으로 흐름 시연; 에이전트는 셋팅만 하고 모델을 직접 실행하지는 않음).
3. 여러 시나리오를 큐로 자동 수행 → 우측 3D 뷰에서 부산권(마린시티·해운대) 침수 결과를 비교.

## 효과 (자기 업무에 맞는 지표로 — 형식 자유)

> **핵심**: 기존에는 10단계 폴더마다 들어가 **특정 변수(단층 케이스·SSP·기간·해상도 등)를 직접 바꿔** 실행해야 했지만, 지금은 **웹 플랫폼 GUI에서 변수를 설정**하거나 **에이전트에게 자연어로 변수 설정을 맡겨** 동일 작업을 한다. 수치모델 수행(계산) 시간 자체는 동일해 시간을 정량적으로 단축했다고 단정하긴 어렵지만, **이 방식 전환으로 폴더별 반복 수작업이 사라져 훨씬 효율적**이다. (정성 위주)

| 지표 | Before (수작업) | After (에이전트화) | 효과 |
|------|--------|-------|------|
| **케이스 전환 절차** | 단계 폴더 10개를 하나하나 들어가 파일·경로·케이스번호를 직접 열어 수정하고 순차 실행 | 파라미터 1개만 바꾸면 전 단계 설정에 자동 반영 | 폴더별 수동 반복 제거 |
| **파라미터·변수 설정** | 스크립트/텍스트 파일을 직접 편집(유효값을 알아야 하고 오기입 위험) | 좌측 폼·에이전트 채팅으로 설정(검증 포함) | 설정 간소화·실수↓ |
| **다중 시나리오 생성** | 케이스마다 작업자가 직접 값을 바꿔 반복 수행 | 여러 시나리오를 큐에 자동 생성 → 순차 자동수행 | 반복 작업 자동화 |
| **데이터 시각화 고도화** | 단계 폴더에 흩어진 수치/격자 파일을 수동 수집·해석 | 부산 디지털트윈 3D 침수(침수심별 색)·2D 최대수위(maxele)를 한 화면에서 | 결과 이해·비교 직관화 |

### 정성적 효과 (요약)
정량적 시간 단축을 단정하기는 어렵지만, **기존에는 단계 폴더를 하나하나 들어가 파일을 열고 수정하고 모델을 실행하는 한 시나리오의 과정을 케이스마다 반복**해야 해 번거로웠던 반면, 지금은 **설정 → (자동) 수행 → 시각화**로 이어져 **반복·수작업이 줄고 훨씬 효율적**으로 시나리오를 다룰 수 있다.

### jin (전체 통합 · UX/UI · 에이전트 · 부산 디지털트윈)

| 지표 | Before (수작업) | After (에이전트화) | 효과 |
|------|--------|-------|------|
| **결과 해석·전달** | 수치/격자 파일은 비전문가가 해석하기 어려움 | 부산 해운대권 **3D 디지털트윈 침수 시각화(침수심별 색)** + 2D maxele 지도 | 의사결정·발표 임팩트↑ |
| **시각화 자료 정합** | 위성·지형·건물 출처·영역이 제각각이라 위치가 어긋남 | **단일 bbox 재현 베이크**(ESRI 위성·AWS Terrarium DEM·OSM 건물, `npm run data:all`) | 픽셀 정합·재현성 확보 |
| **다중 시나리오 실험** | 결과가 폴더에 흩어져 비교 번거로움 | 좌측 설정 → 큐 **순차 자동수행** → 3D·2D로 한 화면에서(생성 시나리오 저장·재조회) | 일관성·비교 속도↑ |
| **파라미터 설정 접근성** | 텍스트/스크립트 직접 편집 | **에이전트 채팅**(자연어 → 파라미터 설정·스윕 자동 큐) | 비전문가 접근성↑ |
| **시뮬레이션 현실성** | 임의 합성 수식 | 실제 **DEM 기반 침수 판정**·GPU 물 셰이더(곡선 파면·쓰나미 기준 침수색) | 신뢰도·현실감↑ |
| **품질 보증** | 수동 확인 | 전문가 5인 리뷰(서브에이전트) + Playwright 전기능 테스트(콘솔에러 0) | 완성도·재현 검증 |

> 정성 요약: 수치모델 결과(또는 Mock) 위에 **부산 국지 디지털트윈**을 얹어, 좌측에서 설정한 시나리오가 곧바로 3D·2D·비교로 표출되고, 모든 산출물이 **스크립트로 재현 가능**(어느 PC에서든 git clone → `npm install` → `npm run dev`)하도록 통합했다.
