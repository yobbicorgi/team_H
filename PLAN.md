## 1. 한 줄 정의
지진해일(ADCIRC+SWAN) 수치모델 수행을 위한 **10단계 전처리·실행준비 과정**을, 웹에서 **파라미터만 설정하면 다중 시나리오를 자동 수행**하고 **부산권 침수 결과를 3D로 비교**하는 플랫폼.

**궁극 목적:** 기존 10단계의 단순 반복·수작업 자동화.

---

## 2. 기존(Before)의 핵심 문제 — 분석으로 확인된 사실
- 10개 단계(`01_Deform_Plane` ~ `10_SLR_RUN`)가 모두 **Python 스크립트 + 입력데이터(fort.13/fort.14, .dat, .xlsx)** 로 구성.
- 각 스크립트의 입력/출력/파라미터가 **하드코딩**(예: `/home/hlhan/2026/Port/...` 리눅스 절대경로)되어 있음.
- 시나리오를 바꾸려면 스크립트를 **통째로 복사해 `Config` 값만 바꾼 사본**(`_2/_3/_4`, `_7.0`, `_last`)을 만들어 손으로 실행.
- 결과 분기 차원: **지역(Jindo_Wando/Jeju_south…) × SSP(2.6/4.5/7.0/8.5) × near/far × 지진해일 case**.
- → 시나리오 수만큼 "폴더 진입 → 코드 수정 → 실행 → 다음 단계로 수동 연결"을 반복. 번거롭고 실수 위험·일관성 저하.

**자동화의 본질:** 단계별 `Config`(경로·파라미터)를 **한 곳에서 파라미터로 받아 생성**하고, **10단계를 순서대로 오케스트레이션**하는 것.

│    (모델 직접 실행 X, '셋팅값' 설정만)      │    + 정밀 지형고도                │
│  - [실행] 버튼 · 시나리오 큐 · 진행상태     │  - 침수 시뮬레이션 결과 오버레이  │

└───────────────┬──────────────────────────────────────────┬───────────────────┘
                │ REST/JSON (API 계약)                       │ 결과(grid/geojson)
                ▼                                            ▼
┌──────────────────────── Orchestration Backend (Python) ───────────────────────┐
│  ParamSchema → 단계별 Config 생성 → Pipeline Engine(10단계 순차 실행)          │

## 7. Git 협업 워크플로우 (3인 공유 저장소)
> 저장소 `github.com/yobbicorgi/team_H` 를 3인이 함께 사용. 운영 가이드 원칙: **브랜치 안 쓰고 전부 `main`**, **각자 본인 이름 파일만 작업하면 충돌 0**.

### 파일 소유 규칙(충돌 방지)
| 구분 | 파일/폴더 | 소유·작업 방식 |
|---|---|---|
| **개인별** | `submit/teamH_<이름>_PROCESS_LOG.md` | 각자 **본인 것만** 작성 (jin/han/kim) → 충돌 0 |
| **공통(공유)** | `submit/BEFORE_AFTER.md`, `submit/CHECKLIST.md`, `submit/assets/`, `submit/evidence/`, `PLAN.md`, `README.md` | 공유 산출물. 되도록 **단계별 담당자 1명이 갱신**하고 바로 push. 공동 편집 시 아래 절차 준수 |
| **코드(공통, 디렉터리 분리)** | `backend/`(Jin) · `app/`(Han) · `viz/`(Kim) | 역할=디렉터리로 분리 → 서로 다른 파일을 만져 **충돌 최소화** |

> 핵심: PROCESS_LOG만 개인별, **그 외 submit·코드는 전부 공통으로 push해 공유**. 충돌을 줄이려고 (1) 로그는 이름별 파일, (2) 코드는 역할별 폴더, (3) 공통 문서는 담당자 위주 갱신.

### 일상 루틴 (각자, 반복)
```bash
git pull              # ① 작업 전 항상 최신부터 받기 (남이 올린 공통파일 반영)
# ② 본인 폴더/파일에서 작업
git add -A
git commit -m "한 줄 설명"
git push              # ③ main 으로 바로 올림 (공유)
```
- push가 `rejected` 되면: `git pull` 먼저 → 다시 `git push`.
- 공통 문서를 동시에 고쳐 충돌이 나면 무리하지 말고 담당자/에이전트에게 "충돌 해결해줘"로 위임.
